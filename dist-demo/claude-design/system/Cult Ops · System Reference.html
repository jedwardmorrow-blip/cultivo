Rules · the things to enforce
Read these before designing a new screen. Enforce them in code review.
Polarity
Operator stays sans. Public uses serif. Never both on one surface.
Plex Sans on every operator screen, Plex Serif only on public-facing surfaces (COA, partnership covers, lab reports). The polarity is the point — operator surface is for working, public surface is for reading. The moment a serif lands in the operator app, the system feels "designed for a buyer," and the worker stops trusting it.
Stage colors
Markers, never decoration. Six pixels, full stop.
Stage colors live on stage-dots beside section labels and on the 2-pixel left-bar of harvest-queue pills. Never on borders, fills, backgrounds, status, chart lines, or button colors. If a screen has more than ten stage-color dots, it's wrong.
Single accent
Warm white, alone. The discipline is the brand.
#E8E0D4 drives wordmark, primary buttons, focus, today's highlight, and sparkline closing dots. Status colors handle alarms (ok / warn / bad). Stage colors mark phases. Beyond that — no second brand color, no gradients, no decorative tints. Restraint applied consistently is what reads as "instrument-grade."
Density
Set by the task, not the user. Same DNA, different edits.
A worker on a tablet sees three big buttons (88px tap targets, plain English). A COO at a desk sees the full ledger (Bloomberg-grade rows, mono numbers, every panel showing a footer). Same product, different edits per surface. Don't dial down density to "feel cleaner" — you're stripping signal the operator needs.
Hairlines, not air
Rhythm comes from 1px lines, not whitespace.
The whole operator app is one frame divided by hairlines. Cards-on-cards with shadows is the wrong vocabulary — it reads as "premium SaaS." Stick to flat panels with --op-line dividers and --op-line-strong on section boundaries. Padding stays small; the grid does the work.
Numbers are mono
Plex Mono for codes, IDs, timestamps. Plex Sans for headlines.
Codes (FLR-02, B-2046), IDs, timestamps (11:42:08 PT), targets (1.0–1.4 kPa) all sit in mono — they need to scan-grep cleanly down a column. Headline KPI numbers (4,820, 482.6) stay in Plex Sans 700 — they're text, not data. The split keeps tables readable and headlines authoritative.
Footer signals life
Every panel gets a head and a foot. The foot proves the system is alive.
Panel-foot carries a summary (counts) and an updated timestamp. updated 11:42:08 is the most important text on a screen — it's what tells the operator the data is fresh. Bloomberg has it. Refinitiv has it. Cult Ops has it. Don't skip it to "save space."
Mark usage
The eye, alone or with the wordmark. No effects, no rotation, no color.
Use cult-eye.png at 18px (nav), 22px (auth), 56px (auth hero), 96px (mark study). Always warm white on dark, navy on cream — invert via filter: invert(1) for the public surface. The mark never gets glow, drop-shadow, gradient, or any treatment beyond opacity (0.85–0.95).
Using this kit

Rule for adding a new screen: open the Cultivation Overview, copy the atom or molecule that fits the job, edit only the data. If the job needs something the kit doesn't have, propose it as a new molecule before designing it bespoke. The kit is the contract.

Rule for adding a new molecule: compose it from existing atoms first. If you reach for a new color, font weight, radius, or spacing token — stop. The constraint is the design. Most "I need a new variant" moments resolve to "I'm using the wrong existing molecule."

Rule for the next system version: the kit ships at v1 today. Changes ratchet — every change carries a date and a reason, and we don't add a token without removing one of equivalent weight. The system stays the size of an instrument panel, not a paint catalog.
