**Subheader:**

## SLIDE 3: The CultOps Solution

**Headline:**
**One Platform. Unified Data. Real-Time Visibility.**

**Visual:** Process flow diagram: Cultivation → Post-Production → Inventory → Orders → Delivery (all connected)

**Core Value Proposition:**
CultOps unifies cultivation planning, post-production workflows, inventory tracking, order management, and delivery routing in a single, event-driven platform.

**Three Core Outcomes:**
1. **Eliminate Manual Data Entry**
   Session finalization automatically populates downstream systems. No double-entry.

2. **Real-Time Compliance**
   Native METRC integration ensures compliance documentation is always current. No end-of-day reconciliation.

3. **Operational Intelligence**
   Cost-per-unit analytics, yield analysis, bottleneck identification. Data operators can act on immediately.

## SLIDE 4: How It Works (5-Minute Walkthrough)

**Headline:**
**From Seed to Shipped: Four Core Workflows**

### Workflow 1: Harvest Session
**Screenshot:** CultOps harvest entry screen
**What happens:**
- Cultivator logs harvest: Plant count, trim date, total wet weight
- System auto-creates batch (YYMMDD-STRAIN format)
- Inventory snapshot captured: Wet flower, by batch

### Workflow 2: Trim Session
**Screenshot:** CultOps trim session management
**What happens:**
- Staff enters: Dry weight, bucking time, packaging stage
- System calculates: Yield ratio, cost-per-gram
- Inventory updates: Move from "wet" → "packaged flower" in same batch

### Workflow 3: Order Entry & Fulfillment
**Screenshot:** CultOps order management
**What happens:**
- Sales team enters customer order: Strain, quantity, stage (oz/lb/unit), customer location
- System suggests: Best-available batch, automatic pricing, delivery route
- Order status flows: Draft → Confirmed → Picked → Packed → Shipped

### Workflow 4: Delivery Route Optimization
**Screenshot:** CultOps delivery map + route optimization
**What happens:**
- System generates: Optimal delivery route for multi-stop drop-offs
- Auto-populates: METRC manifests, compliance documentation
- Real-time tracking: Driver updates status; customer notified

**Key Point:**
*All data flows through one system. No manual reconciliation. No data loss between steps.*

## Appendix: Battlecard (For Sales Conversations)

### Objection Handling

**"Why not use BioTrackTHC?"**
> "BioTrackTHC is built for compliance reporting. If you're already compliant with Metrc, you have the foundation. What you need next is **operational visibility**—cost per unit, real-time bottleneck detection, faster order cycles. That's where CultOps wins. Our customers reduce order-to-delivery time from 5+ days to 48–72 hours just by consolidating systems."

**"What about [other tool]?"**
> "Most tools focus on one area: retail (LeafLogix), compliance (BioTrackTHC), or a single function (harvest tracking, COA management). CultOps is the only unified platform that connects cultivation planning → post-production → inventory → orders → delivery. That means one login, one data model, zero manual reconciliation."

**"We're already using [fragmented tools]. Migration sounds risky."**
> "We've built migration as a first-class service. You pick the data you want to move; we handle the rest. Most operators see ROI within 60–90 days from manual-entry time savings alone. And because we're built on modern tech (REST APIs, Supabase), you're not locked in—you can always export your data."
