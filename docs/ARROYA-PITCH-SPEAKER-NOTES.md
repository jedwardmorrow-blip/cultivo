## Slide 4: How CultOps Works (Live Demo / Walkthrough)

**Visuals:**
- 3 key workflows:
  1. Harvest session (plant group → harvest → batch creation)
  2. Inventory movement (batch → trim → package → order allocation)
  3. Order-to-delivery (customer order → batch selection → delivery)

**Text on Slide:**
- Single platform: Cultivation → Post-Production → Inventory → Orders → Delivery
- Real-time data flow: Harvest weight → Batch costing → Order pricing
- Event-driven: Every transaction creates an audit entry (immutable)
- One source of truth: ops, finance, compliance all read from same data

**Speaker Notes (2 minutes):**

"CultOps is built around three core workflows. Let me walk through a real example.

**Workflow 1: Harvest.**
A grower finishes a plant group and logs the harvest. Wet weight is 150 pounds. CultOps automatically:
- Creates a batch (YYMMDD-STRAIN format, e.g., 260401-ZKITTLEZ)
- Allocates production cost (facility overhead, lighting, nutrients, labor)
- Locks the batch for post-production tracking

No spreadsheet. No manual batch creation. No separate METRC entry. One action, all systems updated.

**Workflow 2: Post-Production.**
The batch goes to trim. Ops team logs trim weight, then packaging weight. CultOps:
- Tracks yield loss at each stage
- Recalculates batch cost per remaining unit
- Updates METRC automatically with the inventory reduction
- Flags any yield variance that's outside expected ranges

**Workflow 3: Order Fulfillment.**
A customer wants 10 pounds of Zkittlez. Ops picks the batch, allocates it to the order, and creates a delivery manifest. CultOps:
- Deducts from batch inventory
- Updates order status
- Generates METRC delivery request
- Flows cost of goods into order pricing and then into the accounting system

Everything is connected. Everything is auditable. And everything is real-time.

The magic: event-driven architecture. Every transaction—harvest, trim, packaging, order fulfillment—creates a permanent, immutable log. That means when a regulator audits, there are no gaps, no inconsistencies, no 'we'll have to check with ops.' It's all there."

**Your response:**
"LeafLogix and Metrc-native tools own specific niches. LeafLogix is strong in retail POS, not production. Metrc tools are point solutions—good for compliance, not operations. We are the only unified platform that goes from cultivation to order to delivery. And more importantly, we're the only one with real-time visibility and event-driven audit trails.

## Final Notes for CEO

1. **Know your numbers cold.** Be ready to articulate ROI, TAM, customer metrics without hesitation.
2. **Let the customers speak.** If Arroya asks for references, have 2-3 ready to do calls within 48 hours.
3. **Tie CultOps to Arroya's strategy.** This isn't just a software deal; it's a customer deepening play. Frame it that way.
4. **Be clear on exclusivity scope.** Define whether this is geographic (West Coast?), channel-based (distributions only?), or customer-based (Arroya's existing customers only?). Ambiguity = deal friction.
5. **Own the compliance angle.** This is your strongest differentiator vs. legacy. Compliance = regulators love us = customers sleep at night = Arroya's customers stick with you.
6. **Be realistic about timeline.** First customers took 60-90 days to go live. Set expectations.
